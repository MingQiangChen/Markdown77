# 程序备注：图表、媒体与手写标注

本文记录本轮新增功能的实现说明，方便后续继续开发。

## 1. 涉及文件

主要修改：

- `apps/desktop/src/App.tsx`
- `apps/desktop/src/styles.css`

本轮没有新增后端文件接口，图表、媒体预览和绘图标注均在前端完成。

## 2. 图表渲染

实现位置：

```text
apps/desktop/src/App.tsx
```

核心逻辑：

- 使用 `markdown-it` 的 `fence` renderer 扩展代码块渲染。
- 当代码块语言为 `chart` 时，调用 `parseChartDefinition` 解析内容。
- 解析成功后调用 `renderChartHtml` 输出图表 HTML。
- 解析失败时回退到默认代码块渲染。

当前图表格式：

```text
title: 标题
名称: 数字
名称: 数字
```

设计原因：

- 不增加第三方依赖。
- 不需要联网安装 Chart.js、Mermaid 等库。
- 在老版本 macOS Catalina + Electron 22 环境下更稳。
- 适合 MVP 阶段验证图表编辑和预览流程。

后续升级方向：

- 引入 Mermaid 支持流程图、时序图、甘特图。
- 引入 ECharts 或 Chart.js 支持复杂统计图。
- 支持从 Markdown 表格自动生成图表。

## 3. 多媒体渲染

实现位置：

```text
apps/desktop/src/App.tsx
```

核心逻辑：

- 使用 `markdown-it` 的 `image` renderer 扩展图片渲染。
- 如果 `src` 是视频扩展名，输出 `<video controls>`。
- 如果 `src` 是音频扩展名，输出 `<audio controls>`。
- 其他情况回退到默认图片渲染。

当前支持：

- 图片：沿用 Markdown 默认图片渲染。
- 视频：`mp4`、`webm`、`mov`、`ogg`
- 音频：`mp3`、`wav`、`m4a`、`aac`、`flac`

安全说明：

- Markdown-it 仍然设置 `html: false`。
- 用户手写 HTML 不会直接注入预览区。
- 媒体 renderer 只根据 Markdown 图片语法生成受控标签。

## 4. 手写绘图标注

实现位置：

```text
apps/desktop/src/App.tsx
apps/desktop/src/styles.css
```

核心逻辑：

- 顶部工具栏“绘图”按钮打开 `drawing-panel`。
- 画布使用 `<canvas>`。
- 输入事件使用 Pointer Events。
- `onPointerDown` 开始笔迹。
- `onPointerMove` 绘制线条。
- 点击“插入”后使用 `canvas.toDataURL("image/png")` 生成 PNG Data URL。
- Electron 主进程通过 `vault:saveDrawing` 把 PNG 写入 `attachments/drawings/`。
- 前端将返回的相对路径插入当前编辑位置。
- 浏览器演示模式没有文件系统桥接时，回退到内联 Data URL。

为什么用 Pointer Events：

- 同一套逻辑支持鼠标、触控和手写笔。
- 适合后续平板电脑和移动端迁移。
- 比单独处理 mouse/touch/pen 事件更简单。

当前限制：

- 还没有压感。
- 还没有颜色和笔宽选择。
- 还没有橡皮擦。
- 还没有撤销/重做。
- 还没有附件管理面板。

正式版本建议：

- 绘图已经保存到 Vault 的 `attachments/drawings/` 目录。
- Markdown 中已经插入相对路径。
- 绘图数据保留矢量 stroke，导出时再生成 PNG/SVG。
- 移动端使用平台能力优化 Apple Pencil、安卓手写笔、鸿蒙手写输入。

## 4.1 绘图附件保存接口

实现位置：

```text
apps/desktop/electron/main.ts
apps/desktop/electron/preload.ts
apps/desktop/src/global.d.ts
```

IPC：

```text
vault:saveDrawing
```

参数：

- `vaultPath`：当前 Vault 根目录。
- `preferredName`：建议文件名，不需要带 `.png`。
- `dataUrl`：前端画布生成的 PNG Data URL。

安全处理：

- 只接受 `data:image/png;base64,...` 格式。
- 文件名会清理非法字符。
- 最终路径通过 `resolveVaultPath` 校验，禁止写出 Vault。
- 如果文件重名，会自动追加序号。

## 5. 工具栏插入逻辑

实现函数：

- `insertMarkdownSnippet`
- `insertChartTemplate`
- `insertMediaLink`
- `insertDrawing`

说明：

- 如果 CodeMirror editor view 可用，则插入到当前光标或选区。
- 如果 editor view 不可用，则追加到当前 Markdown 文末。
- 这保证 Electron 运行和浏览器演示模式都能工作。

## 6. 样式说明

新增样式集中在：

- `.chart-block`
- `.media-embed`
- `.drawing-panel`
- `.drawing-canvas`

设计原则：

- 图表和媒体在预览区保持内容化，不做独立浮层。
- 绘图面板使用覆盖层，避免挤压编辑器和预览区。
- 画布设置 `touch-action: none`，防止平板触控绘图时页面滚动干扰笔迹。
