# 插件系统设计

## 是否一开始做插件

不建议 MVP 第一版就做完整插件市场。

可以先做内部命令系统和扩展点，为后续插件做准备。

## 命令系统

核心功能应该注册成命令：

```text
markdown77.newNote
markdown77.openNote
markdown77.renameNote
markdown77.insertLink
markdown77.togglePreview
markdown77.search
```

这样后续插件、快捷键、命令面板都可以复用。

## 插件能力

未来插件可以支持：

- 注册命令
- 注册侧边栏
- 读取当前文件
- 修改当前文件
- 添加 Markdown 渲染扩展
- 添加设置项
- 监听文件变化

## 插件安全

插件系统必须考虑权限。

建议权限：

```text
readVault
writeVault
network
clipboard
shell
```

默认不允许插件直接执行系统命令。

## 插件格式

```text
my-plugin/
  manifest.json
  main.js
  styles.css
```

示例 manifest：

```json
{
  "id": "my-plugin",
  "name": "My Plugin",
  "version": "0.1.0",
  "main": "main.js",
  "permissions": ["readVault"]
}
```

## 插件市场

插件市场应该放到后期。

上线插件市场前需要：

- 插件审核
- 版本管理
- 安全权限
- 崩溃隔离
- 禁用和恢复机制
