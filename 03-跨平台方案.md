# 跨平台方案

## 平台目标

Markdown77 计划覆盖：

- iOS
- Android
- HarmonyOS NEXT
- Windows
- macOS
- Linux

## 推荐优先级

### 第一优先级

- Windows
- macOS
- Linux

桌面端最适合先做，因为 Markdown 编辑、文件夹 Vault、拖拽、快捷键、插件系统都更容易验证。

### 第二优先级

- iOS
- Android

移动端主要用于阅读、轻编辑、日记和快速记录。

### 第三优先级

- HarmonyOS NEXT

HarmonyOS NEXT 需要单独适配，不建议第一阶段就投入太多。

## 桌面方案

推荐 Electron。

优点：

- 成熟稳定。
- 文件系统 API 能力强。
- 社区资源多。
- 容易复用 Web 编辑器。

缺点：

- 应用体积较大。
- 内存占用较高。

## 移动方案

推荐 Capacitor。

优点：

- 复用 Web 技术栈。
- 支持 iOS 和 Android。
- 适合快速验证移动端体验。

需要额外处理：

- 文件访问权限。
- iCloud / Android 文件系统差异。
- 图片选择和附件保存。
- 后台同步限制。

## HarmonyOS NEXT 方案

推荐 ArkTS + ArkUI。

不要假设 Android 代码可以直接复用到 HarmonyOS NEXT。

HarmonyOS NEXT 版本应先做轻量功能：

- 打开 Vault
- 浏览文件
- 编辑 Markdown
- 预览
- 基础同步

插件系统和复杂图谱可以后置。
