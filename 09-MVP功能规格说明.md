# MVP 功能规格说明

本文定义 Markdown77 第一版 MVP 的功能范围。

MVP 的目标不是做完整 Obsidian，而是做出一个能稳定运行的桌面端 Markdown 知识库原型。

## 第一版目标

第一版只做桌面端：

- macOS
- Windows
- Linux

第一版不做移动端，不做 HarmonyOS NEXT，不做插件市场，不做云同步。

## MVP 核心功能

第一版固定做 6 个核心功能：

1. 本地 Vault 文件夹
2. Markdown 编辑器
3. 实时预览
4. 文件树
5. 全文搜索
6. 双向链接 `[[笔记]]`

## 1. 本地 Vault 文件夹

### 功能说明

用户可以选择一个本地文件夹作为 Vault。

Vault 里的 `.md` 文件就是用户的笔记。

示例：

```text
MyVault/
  首页.md
  日记/
    2026-06-03.md
  项目/
    Markdown77.md
  attachments/
    image.png
```

### 基础能力

- 打开本地文件夹作为 Vault。
- 读取 Vault 内的 Markdown 文件。
- 新建 Markdown 文件。
- 重命名 Markdown 文件。
- 删除 Markdown 文件。
- 保存编辑内容到原文件。

### 暂不支持

- 多 Vault 管理。
- Vault 加密。
- Vault 云同步。
- 团队共享 Vault。

## 2. Markdown 编辑器

### 功能说明

提供一个可用的 Markdown 编辑区。

推荐使用：

```text
CodeMirror 6
```

### 基础能力

- 编辑 Markdown 文本。
- 自动保存。
- 支持常用快捷键。
- 支持代码块。
- 支持中文输入。
- 支持撤销和重做。

### 暂不支持

- 所见即所得编辑。
- AI 写作。
- 复杂表格编辑。
- 插件扩展编辑器。

## 3. 实时预览

### 功能说明

编辑 Markdown 时，右侧或下方显示实时预览。

### 基础能力

- 标题
- 段落
- 列表
- 引用
- 代码块
- 图片
- 链接
- 表格

### 推荐解析库

```text
markdown-it
```

或：

```text
unified / remark / rehype
```

MVP 阶段优先选简单稳定的方案。

## 4. 文件树

### 功能说明

左侧显示 Vault 文件树。

### 基础能力

- 展示文件夹。
- 展示 `.md` 文件。
- 点击文件打开编辑。
- 新建文件。
- 新建文件夹。
- 重命名文件。
- 删除文件。

### 暂不支持

- 拖拽排序。
- 文件标签视图。
- 多选批量操作。

## 5. 全文搜索

### 功能说明

支持搜索 Vault 内所有 Markdown 文件。

### MVP 搜索方式

第一版可以先用简单文本搜索，不必一开始上复杂索引。

搜索范围：

- 文件名
- Markdown 正文

搜索结果展示：

- 文件名
- 匹配片段
- 点击后打开文件

### 后续升级

第二阶段可以引入 SQLite 或全文索引。

## 6. 双向链接

### 功能说明

支持 Obsidian 风格链接：

```markdown
[[笔记]]
[[文件夹/笔记]]
[[笔记|显示名称]]
```

### 基础能力

- 在预览中识别 `[[笔记]]`。
- 点击链接打开对应笔记。
- 如果笔记不存在，点击后可以创建。
- 当前笔记显示反向链接列表。

### 反向链接

扫描 Vault 中所有 Markdown 文件，找出引用当前笔记的文件。

示例：

```text
当前笔记：Markdown77.md

反向链接：
  00-项目知识库总览.md
  07-开发路线图.md
```

## 验收标准

MVP 完成时，应该满足：

- 可以打开一个本地文件夹作为 Vault。
- 可以看到左侧文件树。
- 可以新建并编辑 Markdown 文件。
- 可以实时预览 Markdown。
- 可以搜索 Vault 内文件。
- 可以使用 `[[笔记]]` 跳转。
- 可以看到当前笔记的反向链接。
- 关闭应用再打开后，文件内容仍然存在。
