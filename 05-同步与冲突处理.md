# 同步与冲突处理

## 同步原则

同步应该基于普通文件，而不是私有数据库。

优先支持：

- iCloud Drive
- WebDAV
- OneDrive
- Dropbox
- 本地文件夹同步

后期再考虑自研云同步。

## 为什么先支持 WebDAV

WebDAV 的优点：

- 跨平台。
- 国内可用服务较多。
- 不需要深度绑定某个云厂商。
- 适合个人知识库同步。

## iCloud 同步

iOS 和 macOS 用户可以把 Vault 放进 iCloud Drive。

注意：

- iCloud 可能产生冲突副本。
- 多设备同时编辑同一文件有风险。
- 移动端要处理文件未下载到本地的状态。

## 冲突处理

基础策略：

1. 比较本地文件修改时间。
2. 比较远端文件修改时间。
3. 如果双方都修改，生成冲突副本。
4. 不自动覆盖用户内容。

冲突文件命名：

```text
笔记名.conflict-2026-06-03-153000.md
```

## 同步状态

每个文件需要记录：

```text
path
local_mtime
remote_mtime
local_hash
remote_hash
sync_status
last_sync_at
```

## 加密

端到端加密可以作为高级功能。

早期不建议一开始实现，因为会影响：

- 搜索
- 预览
- 同步冲突处理
- 插件读取

可以先设计接口，后期实现。
