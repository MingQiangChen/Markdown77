# 技术架构

## 总体架构

推荐使用“核心引擎 + Web UI + 平台壳”的架构。

```text
核心引擎：
  Markdown 解析
  文件索引
  搜索
  链接分析
  同步
  插件协议

Web UI：
  编辑器
  文件树
  预览
  设置
  插件界面

平台壳：
  桌面端
  移动端
  HarmonyOS NEXT
```

## 推荐技术栈

### 桌面端

优先推荐 Electron。

原因：

- 跨 Windows、macOS、Linux 成熟。
- 文件系统能力强。
- 与 VS Code、Obsidian、Cursor 路线接近。
- 对 Markdown 编辑器非常合适。

备选：Tauri。

Tauri 体积更小、性能更好，但开发复杂度更高，移动端生态也需要额外评估。

### 前端

推荐：

- React
- TypeScript
- CodeMirror 6
- markdown-it 或 unified / remark

### 移动端

推荐先用 Capacitor。

原因：

- 可以复用 Web UI。
- 支持 iOS 和 Android。
- 适合先做移动 MVP。

### HarmonyOS NEXT

建议单独做 ArkTS + ArkUI 适配。

HarmonyOS NEXT 不应被简单当作 Android。它需要单独处理：

- 文件访问
- 权限
- UI 适配
- 应用生命周期
- 打包发布

## 核心包设计

```text
packages/core
  Vault 管理
  文件事件
  配置管理

packages/markdown
  Markdown 解析
  链接解析
  标签解析

packages/search
  全文索引
  查询接口

packages/sync
  同步协议
  冲突处理

packages/plugin-api
  插件注册
  命令系统
  扩展点
```
