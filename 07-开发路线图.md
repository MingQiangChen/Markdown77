# 开发路线图

## 第 0 阶段：产品验证

目标：确认核心体验。

任务：

- 确定目标用户。
- 确定 Vault 文件模型。
- 确定编辑器技术栈。
- 做一个 Web 原型。

## 第 1 阶段：桌面 MVP

目标：做出可用桌面版。

功能：

- 文件夹 Vault
- 文件树
- Markdown 编辑
- Markdown 预览
- 新建、删除、重命名笔记
- 全文搜索
- 双向链接
- 标签识别

平台：

- macOS
- Windows
- Linux

## 第 2 阶段：同步

目标：多设备可用。

功能：

- WebDAV 同步
- iCloud Drive 支持
- 同步状态显示
- 冲突副本
- 同步日志

## 第 3 阶段：移动端

目标：iOS 和 Android 可阅读、可轻编辑。

功能：

- 打开 Vault
- 浏览笔记
- 编辑笔记
- 图片附件
- 基础搜索
- 同步

## 第 4 阶段：HarmonyOS NEXT

目标：完成 HarmonyOS NEXT 单独适配。

技术：

- ArkTS
- ArkUI
- DevEco Studio

功能先保持轻量：

- 文件浏览
- Markdown 编辑
- 预览
- 同步

## 第 5 阶段：高级能力

功能：

- 插件系统
- 图谱视图
- 主题系统
- 发布插件市场
- AI 辅助写作
- 端到端加密

## 风险清单

- 全平台维护成本高。
- 同步冲突复杂。
- 移动端文件权限差异大。
- HarmonyOS NEXT 需要单独投入。
- 插件安全风险高。
- 自研云同步成本高。

## 建议节奏

一个人开发时，优先顺序应该是：

1. 桌面版。
2. WebDAV 同步。
3. 移动端。
4. HarmonyOS NEXT。
5. 插件和 AI。
